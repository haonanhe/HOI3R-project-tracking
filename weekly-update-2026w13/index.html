2 / Trellis-Inspired Route

新主线：轻量 token memory，而不是直接做重头回归

按现有设计文档，这条线的关键不是换 backbone，而是把 hand / scene / object 组织成显式 memory，再只做 residual 更新。

一句话

Frozen STream3R 提供 camera/depth/world scaffold，HaMeR 提供 hand observation base，object 侧拆成 static shape prior + render-conditioned token，最后由 align_hand_net 做 residual-only refinement。

01 / Base observation

HaMeR hand base

  • 先把 crop-camera 输出理顺到 full-image `C_t`。
  • 世界坐标 lift 统一走 `T_{W←C_t}`。
  • 第一阶段只验证“相机接口是否对齐”。
02 / Scene memory

STream3R scene tokens

  • 主干不动，只读取 camera / depth / world scaffold。
  • 先试 full-frame，再收缩到 hand ROI + confidence。
  • Stage-2 当前 winner 是 ROI + confidence 方案。
03 / Object memory

Static + Render 双路 token

  • 借鉴 trellis / Hunyuan 的 object prior 思路。
  • `O_static` 负责 canonical shape prior。
  • `O_render_t` 负责 pose-conditioned visible geometry。
04 / Residual update

align_hand_net

  • 只输出 `Δθ / Δβ / ΔT_{C_t←H}`。
  • 不做 direct regression，避免覆盖基础分支。
  • 后续再接 temporal / infiller，而不是现在就一起上。

5 / Next Two Weeks

接下来不是再开新坑，而是把主线收窄

当前最重要的是把 Stage-2 winner 往 object / tri-modal 延伸，并且补齐 slice-grade 指标，而不是同时开多个高风险分支。

优先级

先 oracle object，再 predicted object；先 residual fusion，再 temporal。 周末继续跑 baseline / long jobs，工作日集中做高认知设计和实验判读。

Stage 3A

先做 oracle object ceiling，验证 object 信息是否真的能帮 hand。

  • 跑 `static-only / render-only / static+render`。
  • 聚焦 contact subset。
  • 形成可视化和小表格。

Stage 3B

再切到 predicted object pose，检查 oracle gain 能保留多少。

  • 上线 `object_render_encoder`。
  • 比较 predicted vs oracle gap。
  • 继续保留 render 视频证据。

Stage 4

推进 tri-modal residual fusion，但 direct regression 只做负对照。

  • memory/query contract 冻结。
  • 对比 full fusion 与 partial models。
  • 避免“模型变重但不清楚谁在起作用”。
