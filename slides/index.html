1 / Background & Motivation

Why End-to-End HOI Reconstruction?

Existing methods tackle hand reconstruction, object pose, and scene geometry separately. This creates error accumulation and misalignment when combining outputs. We propose a unified pipeline that jointly reasons about hands, objects, and scenes.

Problem Setting
Monocular Video → Full 3D
Input: RGB video of hand-object interaction.
Output: scene point clouds, camera poses, hand mesh (L/R), object mesh — all in a shared world coordinate system.
Gap 1

Isolated Pipelines

Hand recon (HaWoR, HaMeR) and object pose (FoundationPose) run independently with no shared scene understanding.

Gap 2

No Joint Scene Context

Without scene point clouds and camera geometry, hand-object spatial relationships cannot be properly resolved.

Gap 3

Coordinate Misalignment

Each method uses its own coordinate space. Post-hoc alignment introduces errors and prevents end-to-end training.

Object-Only Baseline

FoundationPose recovers object pose but has no hand awareness — misses interaction context.

2 / Method Pipeline

Architecture: STream3R + Trellis.2 Fusion

Five-stage pipeline: frozen encoders extract image tokens and mesh latents, the STream3R decoder fuses them via a trainable token bridge, then Trellis.2 decoder outputs posed meshes.

Frozen Encoders

  • STream3R encoder: pretrained 3D scene understanding
  • Trellis.2 SC-Encoder: canonical mesh → Structured Latents
  • No gradient, preserves pretrained representations

Token Bridge

  • Projects SLat into STream3R token space
  • Cross-modal attention in decoder global blocks
  • Only ~66K parameters — lightweight fusion

Output Contract

  • All outputs in STream3R world coordinate W
  • W = first-frame camera coordinate system
  • Hand/object mesh directly in W, no post-hoc alignment
