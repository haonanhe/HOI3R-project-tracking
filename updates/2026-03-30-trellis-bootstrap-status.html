DexYCB Data Contract

While the raw dexycbshu archive is still extracting, the code path is already clear: training does not read the raw directory directly. It expects the processed output produced by datasets_preprocess/preprocess_dexycb.py.

  • Processed root provides manifest.json and optional split_lists/.
  • Sequence payloads are frame-wise .npz records carrying camera intrinsics/extrinsics, MANO pose/betas/translations, and object pose / mesh references.
  • DexYCB_Multi resolves single-sequence filtering through the processed manifest, so the real overfit path can be switched on as soon as preprocessing output appears.
