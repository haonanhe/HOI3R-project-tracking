7. Analysis: Why Chamfer Doesn't Improve

VALIDATED Token bridge learns to preserve SLat

Latent reconstruction loss: 1023 → 0.10 (99.99%). The token bridge successfully learns the 32→1024→32 roundtrip through STream3R attention. This is the intended Phase 2 objective.

ROOT CAUSE Gradient flow does not reach the decoder

The latent reconstruction loss compares refined_latents (output of token_bridge.output_proj) against encoder_latents.detach(). The gradient path is:

loss = L2(refined_latents, encoder_latents.detach())
         │
         ▼
  token_bridge.output_proj  ← receives gradient ✓
         │
         ▼
  aggregator.global_blocks  ← receives gradient ✓
         │
         ▼
  token_bridge.input_proj   ← receives gradient ✓
         │
         ▼
  shape_slat_encoder        ← frozen, gradient stops here

  shape_slat_decoder        ← NOT in this gradient path at all ✗
  (decoder runs AFTER output_proj, but loss is computed BEFORE decoder)

The decoder produces meshes, but the loss is on latent features before the decoder. Even though the decoder has requires_grad=True, it never participates in the backward pass. Its 474M parameters are effectively frozen.

CONSEQUENCE Chamfer is bounded by encoder roundtrip fidelity

Since only the token bridge is trained, the best achievable mesh is what the frozen encoder+decoder produce in a direct roundtrip (latent_recon=0 means perfect bridge, so decoder input = encoder output). The pre-optimization chamfer of Exp B (hand=0.0054, object=0.0095 from the Phase 2 validation) already represents this ceiling — the Trellis.2 encoder→decoder lossy compression limit. Training the bridge can only recover this ceiling, never exceed it.

OBSERVATION Geo loss (monitor) is also flat

The render-based geometry loss (hand=0.017, object=0.024) stays constant from step 1 to step 500, confirming that mesh quality doesn't change. This is consistent with the chamfer plateau.

8. What Needs to Change

To actually improve mesh reconstruction quality beyond the encoder roundtrip ceiling, we need loss gradients to flow through the decoder:

ApproachGradient pathWhat it optimizes
A. Chamfer loss through decoder chamfer(pred_mesh, gt_mesh) → decoder convs → latent features → token bridge Decoder learns to produce meshes closer to GT
B. Render loss through decoder L1(rendered_depth, gt_depth) → differentiable renderer → decoder → latent → bridge Decoder learns to produce meshes that render correctly
C. Subdivision + intersected supervision BCE(subs_pred, subs_gt) → decoder upsample blocks Decoder learns correct spatial structure (already wired, needs activation)

The decoder training-mode path (calling SparseUnetVaeDecoder.forward to get subs_gt/subs) and the explicit_gemm backward fix are already implemented. The missing piece is a loss term that flows through the decoder output to the decoder parameters.
