Key discoveries

FINDING 4RC encoder attention makes hand↔object contact free.

4RC ViT-G uses alternating frame-wise + global self-attn with a single token pool per layer. If hand/object query tokens are added to the encoder input alongside patches, they (a) attend to each other per-frame (contact reasoning) and (b) attend cross-frame (trajectory correspondence) automatically. This is the architectural lever that made V2-sampled (queries in encoder) strictly better than V1 (queries in decoder only) for our HOI task.

FINDING NLF's "post-fit" is query-at-inference, not Adam.

NLF does NOT perform SMPLify-style Adam fitting at inference. The network takes arbitrary canonical coords as input and outputs posed coords directly via model.get_weights_for_canonical_points. For HOI3R V2-sampled, this means training with 128 hand + 256 obj barycentric samples (token-budget manageable) and at inference querying with all 778 hand + full object canonical verts — no parametric fit. Preserves feedback_dcf_only_points_principle.md (no MANO/SE(3) regression).

FINDING Existing feat-dcf dcf_query_sampling.py already enforces sample consistency across frames.

The user's explicit constraint — "within each step, hand/object samples must match across all frames in the clip" — is structurally required for motion-head correspondence. sample_query_batch returns (bary_coords, face_indices) once per call; warp_points_to_posed applies the same indices to each frame's posed vertices. We reuse this verbatim in the new DexYCB16FrameClip loader.
