Experiment Setting

Synthetic migrated overfit

A short migrated synthetic replay was exported with both pre-optimization and post-optimization compare views. The setting is useful as a clean contract check because the scene is controlled and the expected geometry path is already understood from the earlier synthetic bootstrap work.

Purpose

Check migration wiring before claiming training gain

The purpose here is to verify that the migrated STream3R-to-TRELLIS path still produces interpretable overlay and world-space outputs before and after a minimal replay. This setting is not used to argue convergence; it is used to show that the structural path remains stable after migration.

Results

Geometry path stays consistent, numeric change is negligible

The copied pre/post overlay and world-space compares remain on the same geometric path, which is the main evidence that the migrated backbone and TRELLIS branch still connect coherently. The reported synthetic loss moved from 0.09989190101623535 to 0.09989138692617416, a delta of 5.140900611877441e-07.

Conclusion

Migration proof, not optimization proof

The synthetic setting supports the migration claim because the public compare assets remain structurally sensible before and after replay. The loss change is essentially flat, so this should be read as a wiring validation and artifact publication step, not as evidence of meaningful optimization progress.

Experiment Setting

DexYCB frame30 migrated overfit

A short replay was exported for the migrated frame-30 setup with matching pre/post overlay and world-space compare images. This is the more important public setting because it exercises the real-data path rather than the synthetic bootstrap path.

Purpose

Verify real-data structural correctness after migration

The purpose is to confirm that the migrated STream3R backbone and TRELLIS branch still produce interpretable real-scene compares before optimization, then to measure whether a short replay gives any numeric improvement without overstating what that means.

Results

Small numeric gain with stable pre/post public evidence

The published frame-30 compare images show that the migrated path remains reviewable both before and after replay. The reported loss moved from 0.05936773866415024 to 0.05855463445186615, a delta of 0.0008131042122840881. That is a real but small improvement over the short replay window.

Conclusion

Encouraging direction, not convergence claim

Frame30 provides the stronger migration proof because the real-data overlay and world-space paths remain intact through the copied public evidence. The loss improves slightly, but the magnitude is too small to describe as convergence. This node should be read as a structural validation plus a modest positive signal.
