Invariant (load-bearing): each GT loader emits joints/vertices in the convention its dataset's official evaluation toolkit publishes metrics against. Not "everything uses manopth." Concretely: DexYCB → manopth (matches dex-ycb-toolkit), HO3D → manopth, HOT3D → smplx with use_pca=True, num_pca_comps=15. Adapters and metrics consume a single canonical 21-joint layout (dex-ycb-toolkit / OpenPose-style: 0=wrist, 1–4=thumb, 5–8=index, 9–12=middle, 13–16=ring, 17–20=pinky), converted at the loader boundary.

Key discoveries

FINDING HOT3D loader was silently broken on this container.

The JSONL-only hand-trajectory reader produced NaN hand joints for every frame on /mnt/afs/haonan/datasets/haonan/datasets/hot3d/processed/P0001/10a27bf7/214-1/ because the JSONL files do not exist — only per-frame NPZs do. Because the caller never ran downstream checks on the joints, the failure was invisible. The new test flips this into a loud failure mode (and now passes via the NPZ fallback).

FINDING HOT3D's 21-joint layout is already canonical.

After the HOT3D toolkit's own mano_joint_mapping remap is applied (as the loader does), the output is in the same order as dex-ycb-toolkit / OpenPose canonical. So Convention.HOT3D → CANONICAL is the identity permutation — a worthwhile explicit constant for downstream adapters.

FIX Smoke tests on HOT3D must cap at a handful of frames.

Serial CPU MANO FK on the full sequence took long enough to saturate the debug container with zombie pytest processes from repeated timeouts. max_frames=5 on load_sequence_gt and _load_processed_frame_index is now the invariant for any smoke run; Track E smoke scripts (T10–T12) should plumb a smoke_5frame.yaml config key through to it.
