Key discoveries

FINDING HOI3R training forward is 22% FASTER than 4RC pristine inference

On H100 80GB, same 16-frame DexYCB clip (subject-01/20200709_141754 cam 836212060125), bf16-mixed, bs=1, cache DISABLED: HOI3R fwd 611.4 ± 3.0 ms vs 4RC inference 782.2 ± 18.5 ms. Ratio 0.78×. The user's concern that training forward was abnormally slow is not borne out — any per-step slowness lives in backward + optimizer.step() + DDP all-reduce + dataloader, not the forward graph. The 4RC inference wrapper is actually heavier because it runs loss_of_one_batch + CPU movement in addition to the model forward.

FAIL DexYCB left-hand sequences train with right-hand GPS features

DexYCB on disk is 499 left / 501 right sequences. stream3r/data/dexycb_dcf_dataset.py:562 already computes slot_index per side. But hoi3r/data/dexycb_clip_loader.py:72-83 (DexYCB16FrameClip.__getitem__) DROPS that information and returns a single-hand tensor. ArcEncoderWrapper.forward (encoder_wrapper.py:89) adds one scalar self.e_hand to every hand token regardless of side. mano_surface_gps_k16.npy was precomputed on the right-hand MANO layer. Net: ~50% of DexYCB training data is fed the wrong-handed eigenbasis. All Stage-1 H1–H4 + Stage-2 smoke results to date were produced under this bug.

FINDING Handedness architecture decision: 2 fixed slots (left + right)

Evaluated 3 options (A: 2 slots, B: 1 slot + handedness embedding, C: variable-slot set transformer) against param count / efficiency / code complexity / dataset compat / inference / 4RC surgery. Picked A because it's the only option that natively represents 'both hands present' — required for HOT3D bimanual Stage 2. DexYCB pays ~5-10% wasted attention on the masked slot (50/50 L/R split means one slot is always masked), trivial vs the cost of migrating B→A later. Decision branch: B defensible only if Stage 2 stays DexYCB-only for the whole month.

FINDING 4RC track-viz style: HSV hue per track, 10-frame sliding window, raw-pixel RGB scene

Read arc/viz/viser_visualizer_track.py line-by-line: scene points use raw pixel RGB decode ((img+1)*127.5)/255, sorted by conf desc, top-percentile filtered. Motion tracks: sliding window default_track_len=10 frames, shape (N_tracks, len_t, 2, 3) flattened for add_line_segments, per-track HSV hue stable across time (rainbow comes from the set, not from time), lines[::10] + conf-weighted subsample. For offline matplotlib port: use LineCollection/Line3DCollection (not per-line ax.plot), override HSV with a distinct solid color for hand/obj trails, bump scene alpha to 1.0 for paper figures.

FIX ACP startup scripts must invoke python directly — NO conda run

First submission pair (pt-geubxejk / pt-whnkc1b5) wrote `conda run -n human3r …` in the startup script and died with `bash: line 11: conda: command not found`, exit 127, ~3s after container start. Template pt-xt073hk1 exports PATH=.../envs/human3r/bin:$PATH which makes `python` resolvable but NOT `conda` (conda itself lives under .../miniconda3/bin). Fix: call /mnt/afs/haonan/miniconda3/envs/human3r/bin/python directly — which is the template's actual pattern. Resubmit (pt-dhb96stn / pt-v6k889xj) succeeded immediately. Rule: when cloning an ACP template, preserve its exact command-invocation pattern, don't 'improve' to conda run.

Caveats

  • Forward-timing result (0.78×) is on 1× H100 at bs=1, T=16, image 512², patch 14. Full DDP training-step wall-clock includes backward + optimizer + all-reduce + dataloader — none of which are measured here.
  • Handedness latent bug invalidates the PER-SIDE interpretation of every Stage-1 H1–H4 and Stage-2 smoke metric produced to date. The aggregate numbers are still meaningful for relative comparisons, but a per-hand error analysis requires a rerun after the fix.
  • 2-slot decision assumes Stage 2 moves to HOT3D. If the plan pivots to DexYCB-only-for-the-month, option B (1 slot + handedness embedding) becomes defensible and is cheaper.
  • 4RC track-viz port assumes pred_i_track / pred_i_conf_track keys in the scene npz. Older smoke npz (pre-exp1_4rc_dexycb) lack them; renderer must degrade silently to 'no scene tracks' with a warning.
  • First pair of ACP jobs (pt-geubxejk / pt-whnkc1b5) is FAILED — ignore them. Real results are pt-dhb96stn (4RC) and pt-v6k889xj (HOI3R).
