Status: All 6 ablation configurations show severe SVD collapse (final SVD < 0.12 vs v6 reference 0.400). The DCF DataLoader code path has a fundamental issue that none of the standard hypothesis tests resolve.
Reference values from v6 (working): Hand SVD s3/s1 = 0.400, hand L2 = 8.5mm, object L2 = 9.0mm @ step 1000 on single sequence.

What This Tells Us

Scene loss is NOT the cause. Exp A (zero scene loss) collapsed identically to all others. Hypotheses B, C, D (scene weight magnitude, gradient clipping, freezing scene heads) all addressed scene loss management — none fixed the SVD collapse.
Canonical mesh choice doesn't matter. Exp E used v6's wrist-centered posed canonical (the same representation v6 used to achieve SVD 0.400). It collapsed to SVD 0.030 — the worst result. This proves the canonical change isn't the cause of the DataLoader failure.
Frame temporal locality isn't the cause. Exp F used video_prob=0.0 for v6-style pure random frame sampling from the full tracked range (40+ frame spread vs default 7-12 frames). Still collapsed to 0.027. Predicting the mean across consecutive frames was a plausible cheat path, but it's not what's happening.
What we ruled out:
  1. Scene gradient dominance (B, C, D)
  2. Scene loss interference (A)
  3. Canonical representation (E)
  4. Frame temporal locality (F)
Remaining unexplained: v6's dcf_full_sequence_train.py achieves SVD 0.400. Our DataLoader-based dcf_dataloader_train.py achieves < 0.12 with the SAME config, model, and checkpoint.

Remaining Hypothesis to Test

The only remaining unique difference between v6 and DataLoader: v6 pre-loads ALL frames to CPU once, DataLoader reloads from disk via PIL each __getitem__. Even though individual loads produce bit-identical tensors, the per-epoch reload flow might trigger something subtle (e.g., file system caching, memory layout, gradient interactions with stateful objects).

Next experiment: Build a DataLoader wrapper around load_full_sequence_dexycb that pre-loads all frames once, then samples from memory per __getitem__. Same training script, same batch shape — only the data source changes.
