Phase 2: Four-Way Pipeline Comparison

Isolating where quality degrades in the HOI3R pipeline

Date:2026-04-08 Dataset:DexYCB frame 30, hand mesh (sealed wrist) Resolution:256 (matching Phase 1) Purpose:Identify which pipeline stage causes quality degradation

Four Paths Compared

PathPipelineLatent TokensPred VerticesStatus
1. Direct Trellis.2 GT mesh → seal → normalize → voxelize → reference encoder → reference decoder 640170,401 Correct
2. Phase2 no bridge GT mesh → _encode_slot → raw latent → decoder (bypass token bridge) 640170,504 Correct
3. Bridge only GT mesh → _encode_slot → input_proj (32→1024) → output_proj (1024→32) → decoder 64053,563 Degraded
4. Full STream3R GT mesh → _encode_slot → input_proj → STream3R global_blocks → output_proj → decoder 64053,735 Degraded (same as #3)
Path 1 vs Path 2: Latent features match with max diff = 0.000000. The Phase 2 _encode_slot (normalization, voxelization, encoding) is correct.
Path 2 vs Path 3: The token bridge (randomly initialized input_proj 32→1024 + output_proj 1024→32) distorts the latent features, causing the decoder to produce a degraded mesh (53K vs 170K vertices). This is expected — the bridge needs training.
Path 3 vs Path 4: Adding STream3R global attention blocks between input_proj and output_proj has minimal additional effect (53,563 vs 53,735 verts). The frozen STream3R global_blocks act as near-identity on these extra tokens. The degradation is entirely from the untrained token bridge projection, not from STream3R.

Conclusions

  • Encoding is correct: Phase 2 _encode_slot produces identical latents to the reference Trellis.2 pipeline (uniform normalization fix applied)
  • Token bridge is the bottleneck: The randomly-initialized linear projections (32→1024→32) destroy latent information. Mean roundtrip error: 0.82 (vs latent std: 5.46)
  • Training target identified: The latent reconstruction loss must train the bridge to be near-identity initially, then allow STream3R attention to inject useful visual features
  • Resolution matters: sparse_resolution=256 produces 640 latent tokens; sparse_resolution=8 collapsed to 1 token (meaningless)
