Root cause: frozen > unfrozen on hand rank

Mechanism: gradient-magnitude balance on shared aggregator.global_blocks.

Scene loss (~268k pixels/view × 4 views × 5 components) dominates DCF hand loss (256 points) by ~20,000:1 in gradient contributions. When scene heads are unfrozen, the heads adapt to absorb scene errors, freeing the backbone to drift into scene-optimal low-rank features. DCF hand tokens inherit this low-rank → hand collapses along its naturally-thin axis (palm normal).

Fix: loss_weight_scene=0.1 (not 1.0). Keeps scene loss active but 10× lower gradient magnitude on backbone. DCF hand tokens retain per-point diversity.

Why frozen “works”: frozen heads constrain the scene-loss gradient to the pretrained Jacobian’s rank — an implicit low-rank regulariser. Unfreezing removes that constraint. Lowering the loss weight is the precise lever; freezing is the coarse lever.

Open issues & next steps

PriorityIssueDetailProposed fix
P0 CUDA index-OOB on cross-subject sequences Some DexYCB sequences trigger IndexKernel.cu assertion in model forward, poisoning GPU context. Makes full-manifest & multi-GPU training unviable. Run with CUDA_LAUNCH_BLOCKING=1 to get exact Python traceback. Fix the indexing bug or filter the triggering sequences.
P0 Hand SVD collapse in multi-sequence loss_weight_scene=0.1 works for single-seq (SVD 0.40) but multi-seq oscillates 0.05–0.22. Each new sequence resets gradient landscape before DCF settles. Try loss_weight_scene=0.01–0.03; or increase steps_per_sequence to 50–100; or warmup schedule (scene_weight decays over training).
P1 DDP crashes on multi-subject Different YCB objects have different canonical mesh vertex counts. DDP requires identical forward graph across ranks. Pad canonical meshes to common max size (8192 vertices + mask). Or restrict to same-object runs.
P1 Static loss_pts3d_grad / loss_depth_grad gradient_loss_multi_scale returns same value regardless of pred. Currently disabled via --scene-grad-mode none. Investigate multi-scale averaging + NEAREST-downsampled pointmap interaction. Low priority since conf terms carry useful signal.
P2 trajectory_3d_loss blind to rank collapse F.l1_loss doesn’t penalize distributional collapse. A line through GT centroid has similar L1 to a 3D cluster. Chamfer recovers rank but breaks per-vertex correspondence. Need l1+chamfer combined (implemented, untested) or separate --hand-loss-type / --obj-loss-type.
P2 Phase-3 full-sequence evaluation All experiments so far are 5-frame smokes or small multi-seq. Need held-out sequence evaluation for generalization metrics. Once multi-seq training is stable, evaluate on DexYCB test split.
