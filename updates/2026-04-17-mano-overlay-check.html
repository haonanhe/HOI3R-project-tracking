2026-04-17 · MANO Gate Visual Check v3 — manopth fix (matches dex-ycb-toolkit)

Branch: feat/benchmark Renderer: benchmark/preprocessing/render_mano_overlay.py Source: /mnt/afs/haonan/datasets/haonan/datasets/{HO3D,dexycb}

Goal: The numeric MANO validation gate passed for HO3D and DexYCB, but a numeric pass only proves NaN-freeness, joint count, and value ranges. This page projects the loader's GT (hand vertices, hand 21-joint skeleton, object vertices) onto the source RGB to verify geometric alignment.

v2 diagnosis — smplx and manopth disagree materially on DexYCB inputs. Running both MANO libraries on the same DexYCB pose_m + betas gave identical wrist and all five MCP joints (they're driven only by shape+translation), but up to 53 mm divergence at PIP / DIP / TIP joints — the libraries interpret the PCA pose basis or flat_hand_mean convention differently. v2 (smplx-based) looked "off" on DexYCB not because of bone topology, but because the 3D joints themselves were wrong. Verification script: /tmp/compare_smplx_vs_manopth.py on the debug container.
v3 fix — port GT loaders to manopth. benchmark/data/dexycb_loader.py and benchmark/data/ho3d_loader.py now use manopth.manolayer.ManoLayer (not smplx) with the exact configs the official dex-ycb-toolkit/examples/visualize_pose.py and the HO3D reference adapter use. Manopth returns 21 joints directly in the toolkit canonical order — no fingertip-vertex trickery, no remap. The renderer's bone topology is now dex_ycb_toolkit._MANO_JOINT_CONNECT verbatim (five contiguous finger chains 0-1-2-3-4, …, 0-17-18-19-20). DexYCB overlays below should now pixel-match the dex-ycb-toolkit section further down the page.

Legend & method

Hand vertices (778 verts) — semi-transparent Hand joints (21) + skeleton bones Object vertices — subsampled to 5k for legibility

Projection: (u, v) = (K @ (X / 1000) / Z)[:2] with X in mm camera-space, K in pixel units (per-frame, shape (T, 3, 3)). Points with Z ≤ 0 or off-image are filtered.

Datasets share the per-frame intrinsics returned by the GT loader (HO3D: 30 frames, DexYCB: 74 frames per chosen test sequence).

Skeleton bones (v3): dex_ycb_toolkit._MANO_JOINT_CONNECT — five contiguous finger chains on the toolkit canonical 21-joint layout (0=wrist, 1-4=thumb, 5-8=index, 9-12=middle, 13-16=ring, 17-20=pinky).
