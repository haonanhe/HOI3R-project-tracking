Goal
Fix the DCF hand-shape collapse reported in the 2026-04-17 morning session (SVD 0.669 → 0.077); understand whether the model is reading images or memorizing the sequence.

TL;DR

Shape collapse in the failed DCF DataLoader overfit was a chain of four independent bugs; fixing all four got the tight 5-frame overfit to 8.7 mm hand L2 — in the v6 baseline regime. The same recipe scaled to the full 63-frame sequence gave 64.1 mm but with a mode-collapse signature in approach frames (chamfer/L2 = 1.20). A targeted black-image ablation confirmed the model DOES use images (pred shifts 80–190 mm when the image is blanked) but is dominated by a memorized grasp-pose prior. Multi-seq DDP + chamfer + object-grouped batching pushed approach chamfer/L2 to 1.06 in ~32× the wall-time throughput of the single-GPU trainer — but the comparison is confounded because chamfer was also added to the loss on that run.

Key Discoveries

D1 — The DCF head has no view dimension by design

FINDING The aggregator concatenates the canonical extra tokens ONCE with the flattened [B, S*P, C] visual sequence and returns the refined extras as [B, N_e, C] (no S). That's why the original per-view supervision bug was so destructive: the head emits one set of DCF predictions per batch element, not per view, so supervising it against a randomly-chosen view's GT (the broken primary=rng.integers(0,V)) trained the head toward the per-point mean across views — a smeared, rank-collapsed hand blob.

Fix (commit 7634342): broadcast extras to [B, S, N_e, C] before global attention, interleave with each view's visual patches so the causal mask applies uniformly over S*(P+N_e) tokens. The DCF head now emits [B, S, N_h, 3] per-view predictions symmetric with the main branch's per-view pose_enc/depth/world_points.

D2 — GPS skip at the trajectory head is required for shape preservation

FINDING The encoder applies Fourier PE on canonical xyz so per-point input identity is strong; but the trajectory head (DCFTrajectoryHead, a 3-layer MLP) has no skip connection without use_gps=True. Under attention mixing the decoded tokens lose per-point identity. GPS gives the head a fixed 16-dim "name tag" per FPS-selected vertex. The 2026-04-14-gps-encoding-experiments page documented this already; neither training script set it to True. Default flipped in commit 16839a2.

D3 — Single-sequence overfit permits memorization shortcut even with per-view supervision

FINDING Per-view supervision eliminates the averaging-over-views collapse, but on a single sequence the (temporal position in window, hand location) mapping is deterministic and memorizable. The black-image ablation (E4) shows the model does read the image, but the memorized grasp-pose prior dominates in approach frames where image signal is weak (hand small/occluded). This is the structural reason multi-seq training is needed, regardless of loss choice.

D4 — Object-grouped batching unlocks real batched forward

PASS The per-sample grad-accum loop existed because samples from different YCB objects have different V_o (6k–18k vertices). ObjectGroupedBatchSampler buckets the manifest by grasp-YCB-id; every batch is homogeneous in V_o so dcf_collate_fn produces real [B, V_o, 3] tensors and the DCF forward runs once per batch. With DDP, wall-time throughput on subject-01 rose ~32×.
