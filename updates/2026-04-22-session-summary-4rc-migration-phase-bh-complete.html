Key discoveries

FINDING Motion head serial loop dominated step-time. The original HOI3RModel.forward ran 32 motion-head calls per step (B·T·2) as a Python for-loop. Batching to a single (B·T, N, 1536) forward dropped that block from ~10–15 s to ~1 s on 8×H100.
FINDING Encoder output can be cached for overfit. With e_hand/e_obj frozen and a fixed clip, hoi3r_stage1_cached.py runs at 0.19 s/step on a single MIG 3g.40gb (vs 25 s/step baseline = 130× speedup). 3000 steps in 9 min.
FINDING Paper-recipe variable-T is the right Stage-2 win. Quadratic global attention cost is the encoder bottleneck; 4RC's uniform T∈[2,18] amortizes this to ~5× faster than fixed T=16. Validated on pt-so45y00y at 4.85 s/step with correct cross-rank shape sync.
FIX DDP + 4RC reentrant checkpoint. DinoV2's global-attn wraps torch.utils.checkpoint.checkpoint when training=True. Under DDP with find_unused_parameters=True, the reentrant backward marks params ready twice ("ls2.gamma has been marked as ready twice"). Fix: static_graph=True + find_unused_parameters=False (EXP_2/3); or switch encoder to eval() mode so the checkpoint wrapper is never entered (Stage 1 perf-fix).
FIX LoRA adapters landed on CPU. EXP_3 crashed with "input module parameters locate in ('cpu', 'cuda')" because LoRALinear created new nn.Linear after model.to(device). Always move the wrapped model to device after adapter wrapping.
FIX bf16 broke torch.cdist. Inside torch.autocast(bfloat16) the chamfer metric's tensors became bf16 and cdist threw "expected scalar type Float but found BFloat16". Cast metric inputs to .float() explicitly.
FIX Viser default camera was wrong scale for DexYCB. 4RC's start_visualization hardcodes client.camera at z=-0.07 m (robot_arm cm-scale). DexYCB scene is at z = 0.3–3.8 m, so the camera sat behind the scene and Playwright captures showed only the WorldAxes triad. Patched to use a meter-scale centroid-anchored camera.
FINDING EXP_COMPARE: LoRA beats full-FT on rotation. Depth metrics both improve 3–4× over pristine; but pristine rot_err 165° → full-FT 124° → LoRA 37.6°. Full-FT's unconstrained backbone drift disturbs the pretrained cam_dec; LoRA's low-rank constraint preserves it. Informs Stage 3 choice to favor LoRA over full.
