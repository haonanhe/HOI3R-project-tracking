1. Setup

First experiment training the full HOI3R pipeline to learn canonical→posed deformation:

Encoder input: canonical T-pose hand (MANO pose=0, shape=0)
          ↓
    SLat encoder (frozen) → canonical SLat (631 tokens, 32-dim)
          ↓
    input_proj (32 → 1024)          ← trained
          ↓
    global_blocks (+image tokens)     ← trained (302M params, attends to per-frame image)
          ↓
    output_proj (1024 → 32)          ← trained
          ↓
    SLat decoder → predicted mesh    ← trained (474M params)
          ↓
    chamfer_loss(pred, posed_GT)       ← posed w/o global orient

The full pipeline learns to deform canonical mesh toward posed mesh. Chamfer decreases steadily from 0.0665 to 0.0567 (best), a 14.7% reduction. The global_blocks extract pose information from the image and modify canonical SLat tokens to produce posed geometry. This validates the core HOI3R architectural hypothesis.
Oscillation after step 370. CD rebounds from 0.057 to 0.061 by step 500, suggesting learning rate may be too high or the model is overfitting the single training sample. A learning rate schedule or multi-sample training would likely help.

4. Analysis

VALIDATED Full pipeline can learn deformation

The global_blocks + input_proj/output_proj + SLat decoder jointly learn to transform canonical SLat toward posed mesh. Chamfer loss flows through the decoder to all trainable modules. This is the first successful deformation training in the HOI3R pipeline.

OBSERVATION Slow convergence, 4s/step

Each step takes ~4s because the full STream3R forward pass (patch_embed + frame_blocks + global_blocks) runs every step, even though only global_blocks is trainable. Optimizations for future runs:

  • Precompute canonical SLat once (skip encoder every step) — already done
  • Cache image patch tokens from frozen frame_blocks (deterministic) — only run global_blocks each step
  • Gradient checkpointing on global_blocks to reduce memory

NEXT Multi-frame training + optimization

Current experiment overfits a single frame. Next steps:

  • Train on multiple frames from the same sequence (different poses)
  • Implement frame_blocks token caching for 10x+ speedup
  • Add learning rate schedule to reduce oscillation
  • Test Exp 2b (canonical → posed with global orient)
