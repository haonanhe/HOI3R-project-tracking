Key discoveries

FINDING Default PyPI from the debug container is ~1000× slower than Tuna.

Observed PyPI wheel downloads at 10–20 KB/s (scipy 38 MB → 52 min). pypi.tuna.tsinghua.edu.cn/simple serves the same wheels at 91 MB/s. Set globally with pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple. PyTorch CUDA variants: use phased install — pytorch index only for torch, tuna for the rest. Persisted in reference_pypi_tuna_mirror.md.

FIX SSH to the debug container drops on long-running commands.

Foreground SSH wrappers on 30–60 min pip installs dropped multiple times despite ServerAliveInterval=10. Pattern that works: ssh -p 46682 root@... "nohup bash -c '<cmd>' > /tmp/x.log 2>&1 & echo PID=\$!" — install survives the SSH drop, poll via short sessions. Persisted in feedback_ssh_long_commands_drop.md.

FIX HaWoR env needs multiple workarounds.

GitHub clones for pytorch3d@stable and chumpy fail on GnuTLS from this container; use the PyTorch3D py310_cu117_pyt1131 prebuilt wheel and the PyPI chumpy release. chumpy 0.70 still hits ImportError: cannot import name 'bool' from 'numpy'; patch chumpy/__init__.py:11 to import only nan, inf. Lightning 2.2.4 needs setuptools<70 for pkg_resources.declare_namespace. mmcv==1.3.9 and torch-scatter only serve DROID-SLAM/Metric3D (egocentric) — skip. Persisted in reference_hawor_env_quirks.md.

FIX MagicHOI's kaolin needs numpy<2.

kaolin 0.15.0 was built against numpy 1.x; any upgrade to numpy 2.x trips ValueError: numpy.dtype size changed (expected 96, got 88) in triangle_hash.pyx. opencv-python-headless 4.13 declares numpy>=2 but works fine against 1.26.4 at runtime — ignore the pip warning, re-pin numpy<2 after any sweep. Persisted in reference_magichoi_numpy_pin.md.

FAIL FoundationPose weights on Google Drive — not auto-downloadable.

FP refiner (2023-10-28-18-33-37) and scorer (2024-01-11-20-02-45) checkpoints live in a public Google Drive folder. wget/curl from the container do not get an authenticated session, and the Drive CLI isn't installed. Workflow: manually download on a Drive-authed machine, scp -r to third_party/FoundationPose/weights/, then run build_all_conda.sh. Persisted in reference_fp_weights_gdrive.md.

FINDING Existing third_party/ is gitignored on this project.

Adding manopth as a submodule would break pattern. Instead: keep the existing benchmarks_hoi/H2OTR/manopth source and add it to human3r via pip install -e. Removes all sys.path.insert hacks from loaders without altering the repo-tracking convention. T8 commit 0b11472 encodes this choice.

FINDING Dead requirements.txt entries — HaWoR hawor.models does not exist.

The HaWoR repo's top-level hawor/ has configs/ + utils/ but no models/. Correct public API is from hawor.utils.process import run_mano, run_mano_left, get_mano_faces (already how demo.py imports). Test smokes in smoke_run.sh use that path.
