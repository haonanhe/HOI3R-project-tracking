Key discoveries

FIX ACP image torch 2.3.0 pollutes conda env cu118 via LD_LIBRARY_PATH

ACP base image seeds LD_LIBRARY_PATH with /usr/local/lib/python3.10/dist-packages/torch/lib. Conda env cu118 python 3.9 loads host libtorch (wrong ABI) at import torch → segfault in torch::onnx::initONNXBindings. Fix: unset PYTHONPATH PYTHONHOME LD_PRELOAD + strip dist-packages/site-packages from LD_LIBRARY_PATH before conda activate. Confirmed after 9 ACP iterations (v3..v8 segfaulted, v9 SUCCEEDED).

FIX pytorch-lightning 2.3.0 requires torch 2.1+ (LRScheduler API)

MagicHOI's requirements.txt pins pytorch-lightning==2.3.0 but the env's torch 2.0.1+cu118 doesn't have torch.optim.lr_scheduler.LRScheduler (added in torch 2.1). Downgrade pl to 2.0.9 + lightning-utilities 0.9.0 + lightning-fabric 2.0.9 + torchmetrics<1.0 — and install all with --no-deps to stop pl pulling torch 2.8.

FIX Synthesized MagicHOI colmap tree from HO3D GT bypasses COLMAP entirely

HO3D's per-frame GT (K, obj_rot, obj_trans, hand_pose, hand_beta, hand_transl) + YCB canonical mesh is sufficient to construct every file MagicHOI's training reads from the colmap_<seq>.0/ tree: o2w_normalized_aligned.npy, fused.ply / fused_normalized.ply / sparse_points_normalized{_aligned,}.ply, HO3D/cameras/*.json, intrinsic.npy, poses.npy, mat_normalized.npy + hold_fit.slerp.npy + j2d.full.npy + v3d.npy. Replicate MagicHOI's center_and_normalize_point_cloud (rot=I, scale-to-unit-cube). /tmp/synth_magichoi_colmap.py handles it.

FINDING ACP container rootfs + tmpfs are ephemeral; only /mnt/afs persists

Each ACP job start wipes apt-installed packages (libgl1, colmap, libboost-*, libegl1, libosmesa6) and /tmp. Conda envs under /mnt/afs/haonan/miniconda3/envs/ and any path under /mnt/afs/ persists. Every startup script MUST reinstall apt deps + log to /mnt/afs/haonan/tmp/ (not /tmp/).

FINDING Host-download stall for cu118 torch wheel

wget of torch-2.0.1+cu118 (2.15 GB) from pytorch.org from the debug container stalls at ~1.65 GB (80%) indefinitely. Mac curl completes in 2-5 min; scp to /mnt/afs/haonan/tmp/wheels/ is reliable. Document: feedback_acp_wheel_scp.md.

FINDING HaWoR demo.py unconditionally imports DROID-SLAM / lietorch

Earlier memory claimed SLAM was skippable for static cameras — false. demo.py imports hawor_slam at line 8 before any conditional branching. Subagent ae55fa415a5aa4fbe built lietorch 0.3 from source + DROID-SLAM in-place + torch_scatter 2.1.1+pt113cu117 + mmcv 1.6.0 pure-python. Recipe now in reference_hawor_env_quirks.md.
