Key Findings

Finding 1 · Procrustes training as the sole object loss is broken. B.1 eval obj L2 = 779.09 mm (chamfer 1427.22 mm). Kabsch alignment in the loss zeros out the global-rotation and global-translation signal, so the model satisfies training loss while drifting hundreds of mm in world frame. Training-time loss_obj = 0.71 mm was measuring the residual AFTER removing this drift — a textbook "optimised metric ≠ evaluation metric" failure. Never use Procrustes as the sole object-trajectory loss.
Finding 2 · Pose head + consistency (C.00) yields the best object L2 — 13.05 mm. That's 23% better than B.00 (17.03 mm) and passes the <30 mm bar. The learned rigid prior via s · R · canonical + t constrains per-point trajectories to a rigid transform of the canonical mesh; consistency loss couples it bidirectionally to the trajectory head. So the spec hypothesis does hold — the pose head does regularise the object trajectory.
Finding 3 · Pose head breaks hand L2 (+68% on C.00). Hand L2 regresses from 13.98 mm (B.00) to 23.49 mm (C.00), 19.60 mm (C.01), 22.19 mm (C.02). Only C.01 passes the 1.5× hand gate (20.97 mm). The pose head shares the aggregator's global decoder with the hand trajectory head — adding 6 new supervisions there (geodesic R, L1 t, L1 s, CE cls, L1 residual, consistency) pulls capacity away from hand prediction. Candidates for fixing this: reduce pose-loss weights, add a dedicated pose feature path, or schedule pose-head activation after hand converges.
Finding 4 · Stacking pose head + procrustes (C.02) pulls partway back from B.1. C.02 obj L2 = 60.94 mm — bad (43.91 mm worse than B.00) but 12× better than B.1's catastrophic 779.09 mm. The pose head's rigid supervision pulls the trajectory back toward world-frame reasonableness despite the Procrustes loss nulling the global-drift gradient. Interesting as a safety rail but not competitive with C.00.
Finding 5 · Pose head delivers first-class (R, t, s) outputs. On all three C.x legs, training-time rot MAE is ~16° and trans MAE is ~32 mm — directly reportable pose metrics without a post-hoc Kabsch fit. New capability vs B.00/B.1. Pose metrics are still coarse (K = 2048 codebook ≈ 15° Voronoi cell, tanh residual bound 0.35 rad) but sufficient as a first-class model output.

Methodology

Architecture

The DCF branch learns a dense correspondence field between a canonical template mesh and each per-frame observation. This cycle adds an ObjectPoseHead fed by a new learned [CLS_POSE_OBJ] extra-global token appended to the aggregator's token set (contract grows from N_h+N_o to N_h+N_o+1 behind enable_object_pose_head). The head emits codebook+residual rotation (K=2048, seed=0, tanh-bounded residual at 0.35 rad), isotropic softplus scale, and direct translation regression — same parametrization as the main-branch Hand3RObjectBranch, via the shared stream3r/models/so3_codebook.py module.

Supervision

GT (R, t, s) is loaded directly from DexYCB's per-frame object_pose_ct: already in the DCF prediction frame (camera/"ct" frame), with scale = 1.0 by construction. Load-time assertions enforce |s·R·canonical + t − gt_obj_pts| < 1 mm per frame — catches the recurring uniform-scaling bug pattern at the boundary. Six loss terms (weights in parens, default): geodesic R (1.0), L1 t (1.0), L1 s (1.0), cross-entropy on codebook logits (1.0), L1 residual at the GT bin (1.0), bidirectional consistency L1(s·R·canonical + t, pred_traj) (0.5 in C.00/C.02, 0 in C.01). All six are graph-preservingly skipped when object_pose_valid=False.

Ablation design

  • C.00 — pose head + consistency on; tests whether the full stack beats the <30 mm bar.
  • C.01 — pose head on, w_consist=0; isolates the "low-rank prior" signal from the "consistency tying" signal.
  • C.02 — pose head + B.1's Procrustes-aligned trajectory loss (--object-loss-mode procrustes_aligned); tests whether the two object regularizers stack.

Follow-ups

  • Fix the hand-L2 regression (top priority). C.00 has the best object L2 but breaks the hand gate by +68%. Three candidate directions: (a) reduce the six pose-loss weights, especially dcf_pose_cls_weight and dcf_pose_res_weight which pull the aggregator toward discrete rotation classification; (b) add a dedicated pose-feature path so the pose head doesn't share decoder capacity with the hand trajectory head; (c) delay pose-head activation via warmup so hand trajectory converges first.
  • Never use Procrustes as the sole object loss — B.1's 779 mm catastrophe is now documented. The fix is either to combine with a small world-frame L1 weight, or to switch to the pose-head-based rigid loss entirely.
  • Pose accuracy refinement — rotation MAE ~15° reflects the coarse codebook quantization. Options: larger K, multi-step residual refinement, or a second pose-only head with its own feature path.
  • Multi-subject generalization — all five runs trained on subject01 only. Need to confirm C.00's configuration generalises across subjects before it enters the main training path.
  • SAM-3D-Objects integration — per project_sam3d_object_prior_idea, this pose head could become pose-only once object shape comes from a dedicated prior, removing the shared- decoder capacity contention that's hurting hand L2.
  • Methodology note for future runs — always report eval metrics in publish pages, not training-step losses. Training loss_obj is not comparable across object_loss_mode settings.
