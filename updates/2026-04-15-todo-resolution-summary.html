4. Key Discoveries

FINDING STream3R model produces depth/pose even with scene heads frozen

Frozen parameters still run forward; gradients can flow through them into the decoder. This enables real scene supervision without unfreezing scene heads.

FINDING Real scene-loss wiring: +102/−2 lines in one script

Minimal surface because stream3r/loss/utils.py and the bridge dataloader already had most of what was needed. The heavy lift was understanding the correct depth format (mm→m), resolution (518×518 nearest), and intrinsics scaling.

FIX All DCF work in one worktree on feat/dcf

No per-task branches, no integration branches, no sibling worktrees. git worktree remove --force destroys unversioned artifacts (predictions.pt, log dirs) inside — migrate/preserve first or lose them. Jobs #3 and #4 lost artifacts this way.

FIX Smoke-validation ACP protocol: ≤5 frames, one job per feature

Never combine features in one job (if it crashes, can’t tell what broke). Never run smokes on the full sequence (wastes GPU and memory for correctness checks).

FIX Publishing agents must fail loudly on missing artifact dirs

Today’s first republish attempt silently fell back to 500-step historical runs for B and C scatter data — produced a misleading page that had to be torn down and redone.

FIX 3D scatter plot data shape must match reference verbatim

Keys: pred_hand, gt_hand, pred_obj, gt_obj each {x:[],y:[],z:[]}. Colors: Pred Hand #58a6ff, GT Hand #8b949e, Pred Obj #3fb950, GT Obj #6e7681. Improvising key/color names has twice produced swapped/mislabeled plots.
