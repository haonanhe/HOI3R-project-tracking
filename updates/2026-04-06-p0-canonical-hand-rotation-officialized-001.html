Canonical Hand Rotation Officialized

all 5 rotated canonical-hand samples stayed in `142k-181k` decoded-vertex range and did not reproduce the earlier `1.4M+` scattered-hand failure. Falsifies the claim that arbitrary rigid rotation of a canonical hand alone is enough to produce the bad officialized hand result

Context

Experiment goal

直接反证“外部 rigid rotation 本身就足以让 canonical hand 在 officialized path 上变成散乱点云”这个解释。固定旧 `vae_test_initial` 的 canonical hand OBJ，不改形状，只施加 5 个不同刚体旋转，然后继续走与问题案例相同的 officialized `normalize -> dual_grid -> vxz -> encoder -> decoder` 路径，并输出 `GT vs pred point cloud` 可视化。

Conclusion

PASS。这个实验直接推翻了“任意 rigid rotation 足以让 canonical hand 变坏”的简化解释。现在更合理的方向是：bridge hand 和旧 canonical hand 之间存在比整体旋转更深的输入分布差异，这才是后续需要继续定位的主因。

Key findings

  • 5 个样本的 `active_voxels` 分别为 `178,591`, `142,107`, `181,495`, `178,591`, `163,646`
  • 5 个样本的 `pred_vertices` 与 `active_voxels` 一致，范围稳定在 `142k - 181k`
  • 没有一个样本复现之前 `P0-DEXYCB-5SEQ-OFFICIALIZED-001` 中 hand 的 `1.4M+` 级别异常
  • 数值量级与旧 canonical hand repro (`178,591`) 基本一致，说明即使施加明显 rigid rotation，officialized path 仍能稳定处理这个 canonical hand
  • 因此，之前 bridge hand 的坏结果不能仅用“外部旋转相对 voxel grid 不稳定”来解释
