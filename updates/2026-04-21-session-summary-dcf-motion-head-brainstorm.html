Key discoveries

FINDING 4RC is inference-only — no training code released.

GitHub repo Luo-Yihang/4RC (released 2026-04-13) ships inference.py + Gradio app + Arc.from_pretrained(). No train.py, no loss modules, no dataset loaders, no S-4RC (streaming STream3R variant) weights or config. Full backbone migration to 4RC therefore requires reimplementing all four losses (L_depth + L_ray + L_cam + L_motion with aleatoric weighting + temporal gradient) from the paper. Sidesteps this entirely by porting only the 180-LOC MotionDecoder head.

FINDING Factored motion's real benefit for HOI3R is subject-invariance, not temporal consistency.

The motion decoder's queries are canonical-point tokens (identical tensor across subjects); its output is displacement relative to a query frame, not world-space coordinates. This packs subject-specific shape into the trajectory head's per-frame output and leaves the motion module subject-invariant — directly attacking project_overfit_subject01. Temporal consistency benefits (smaller magnitude, ∇tΔP smoothness) are real but secondary.

FINDING 4-view training budget is not a blocker for the dual-head design.

With strict-causal pairs (q<t) the 4-view sample yields 6 forward (q,t) pairs per sample: {(0,1),(0,2),(0,3),(1,2),(1,3),(2,3)}. Motion offsets of 1–3 frames are supervised per sample. Because the trajectory head still produces per-frame absolute points at every t, there is no long-range extrapolation at inference — so the training/inference distribution mismatch that would have plagued a “base+motion” formulation is avoided.
