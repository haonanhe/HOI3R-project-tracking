11. Key Findings

  1. FINDING Object trajectory generalizes perfectly across frames (16mm train ≈ 16mm test, +0.5mm gap). Rigid SE(3) is easy to learn from image features.
  2. CRITICAL Hand trajectory memorizes instead of learning from images (20mm train vs 185mm test, +164mm gap). Even with 72 frames and 2000 steps, hand L2 stays at 190mm. The hand branch ignores image tokens entirely.
  3. CRITICAL No scene losses in training → scene reconstruction collapses. Scene std drops from 1.240 (200 steps) to 0.243 (500 steps). Root cause: shared global attention lets DCF optimization corrupt visual tokens without any scene loss to push back.
  4. WARNING GPS encoding causes complete hand collapse (spread=0) and scene corruption (5x range compression). GPS distillation quality too poor for use.
  5. POSITIVE NLF-style barycentric surface sampling outperforms FPS: 8mm vs 20mm hand L2 in 4-frame overfit. 2.5x improvement.
  6. WARNING Predicted hand points lose geometric structure (collapse to curves). GT warp is correct — the issue is in prediction, not supervision.
  7. FIX Coordinate alignment: GT must be in camera space (STream3R world W), not DexYCB extrinsic world.
  8. FIX DCFBranch head_input_dim=1024 (extra tokens at embed_dim), not 2048 (only visual tokens get 2x concat).

P0 Add STream3R scene losses Root cause of scene collapse. Must add point cloud + camera + depth losses alongside DCF losses to prevent shared attention corruption.
P0 Fix hand geometry collapse Predicted hand points lose geometric structure. Possible fix: predict displacement from canonical position instead of absolute position.
P0 Fix shared attention corruption DCF tokens corrupt visual tokens through shared global attention. Options: freeze global_blocks, detach visual tokens from DCF gradient, or use separate attention streams.

Architecture diagnosis summary: The DCF pipeline fundamentally works for rigid objects (object generalizes perfectly). The hand failure is an attention/conditioning problem, not a representation problem. The hand trajectory head receives image tokens through shared global attention but learns to ignore them because memorization is an easier optimization path for 4-frame overfit. Fixing this requires: (1) scene losses to prevent attention corruption, (2) multi-sequence training to force image dependence, and (3) possibly architectural changes to the hand trajectory head.
