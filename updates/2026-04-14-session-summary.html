1. Architecture: Dynamic Correspondence Fields

DCF replaces Trellis.2 Structured Latents. Instead of voxel-based encoding, canonical hand/object surface points are encoded as tokens, injected into STream3R's decoder via global attention, and decoded to 3D trajectories.

ComponentDescriptionStatus
STream3R encoderPatch embed + frame blocksFrozen (pretrained)
STream3R decoderGlobal attention blocksFrozen (best setting)
Scene headsPoint cloud, camera, depthFrozen
Hand point encoderFourier positional encoding → MLPTrained (random init)
Object point encoderFourier positional encoding → MLPTrained (random init)
Hand trajectory headMLP: decoder tokens → 3D positionsTrained (random init)
Object trajectory headMLP: decoder tokens → 3D positionsTrained (random init)
Contact headCross-attention + MLPTrained (random init)

3. Critical Bugs Found & Fixed

BUG 1 Frames 0,5 have zero hand pose

DexYCB frames 0 and 5 in sequence 20200709-subject-01/20200709_141754 have pose_m all zeros (hand not visible). MANO FK produces T-pose vertices near origin. Including these frames as GT gave artificially low L2 (13mm average = 2mm easy + 24mm real).

Fix: Use --start-frame 10 to skip frames without valid hand pose.

BUG 2 MANO FK already outputs camera-space vertices

_default_hand_mesh_provider() runs MANO FK with global_orient + transl from DexYCB pose_m. The output vertices ARE in camera space (Z≈0.85m). An incorrect "fix" added joint_3d[0] as root translation, double-counting it (Z went to 1.75m).

Fix: hand_verts_ct = hand_mesh.vertices — no root addition needed.

BUG 3 canonical_dim=3 enables shortcut

Concatenating canonical 3D coordinates to the trajectory head MLP allows it to learn pred ≈ R @ canonical + t, bypassing decoder/image features entirely.

Fix: Removed canonical_dim=3. MLP uses decoder tokens only.

BUG 4 GPS distillation too poor for use

GPS encoding (LBO eigenvectors) distillation had val_loss=1978. All canonical points got nearly identical embeddings → hand predictions collapsed (spread=0), scene corrupted via shared attention.

Fix: Deferred GPS, use Fourier encoding as default.

4. Key Findings

FINDING Object generalizes, hand memorizes

Generalization test: object 16mm on both train and test frames. Hand: 20mm train vs 185mm test. Rigid SE(3) is easy; articulated hand deformation is hard.

FINDING Shared attention corruption

Training global_blocks with DCF losses corrupts visual tokens. Scene std drops from 1.24 (pretrained) to 0.24 (500 steps). Freezing global_blocks prevents this.

FINDING Frozen global_blocks is best current approach

Hand 34mm, Obj 11mm, Scene preserved (std 0.98). Predictions DO vary across frames (tracking hand motion), confirming image features are being used despite frozen attention.

FINDING Prediction frame differences match GT

Pred diff between frames closely matches GT diff (correlation >0.96). The model IS using image features to track hand motion, not just memorizing.

FINDING NLF barycentric sampling outperforms FPS

Random surface sampling (240 pts + 16 joints per step) gave 8mm vs FPS 20mm in overfit. Better surface coverage and per-step variation helps, though measured before coord fix.
